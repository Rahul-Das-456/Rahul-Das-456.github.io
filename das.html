-- DAS Spoofing Detection · Flags traders with high cancel rates and directional price impact SELECT o.trader_id, o.symbol, COUNT(*) AS order_count, ROUND(SUM(CASE WHEN o.status = 'CANCELLED' THEN 1.0 ELSE 0 END) / COUNT(*), 3) AS cancel_rate, AVG(o.lifetime_ms) AS avg_lifetime_ms, ROUND(AVG(p.price_impact_bps), 2) AS avg_price_impact_bps, ANY_VALUE(o.side) AS dominant_side FROM surveillance.order_flow o JOIN surveillance.price_impact p ON o.order_id = p.order_id WHERE o.ts >= NOW() - INTERVAL '1 hour' GROUP BY o.trader_id, o.symbol HAVING cancel_rate > 0.85 AND avg_lifetime_ms < 500 AND ABS(avg_price_impact_bps) > 5 AND order_count > 20 ORDER BY cancel_rate DESC;

Flags accounts where cancellation rate exceeds 85%, order lifetime is under 500ms, and the cancelled orders moved the market by more than 5 basis points in the direction beneficial to the trader's remaining fills. The combination of all three conditions is the statutory test under Dodd-Frank §747.

-- DAS Wash Trade Detection · Identifies offsetting trades with zero net Greek change WITH matched AS ( SELECT t1.account_id AS acct_a, t2.account_id AS acct_b, t1.symbol, t1.quantity, t1.price, ABS(t1.ts - t2.ts) AS time_diff_ms, -- Net Greeks after both legs (t1.delta * t1.quantity) + (t2.delta * t2.quantity) AS net_delta, (t1.vega * t1.quantity) + (t2.vega * t2.quantity) AS net_vega, b.common_owner FROM surveillance.trades t1 JOIN surveillance.trades t2 ON t1.symbol = t2.symbol AND t1.quantity = t2.quantity AND t1.side != t2.side AND t1.account_id != t2.account_id LEFT JOIN ref.beneficial_ownership b ON b.acct_a = t1.account_id AND b.acct_b = t2.account_id WHERE time_diff_ms < 1000 ) SELECT * FROM matched WHERE ABS(net_delta) < 0.01 * quantity -- effectively delta-neutral AND ABS(net_vega) < 0.01 * quantity -- effectively vega-neutral ORDER BY time_diff_ms ASC;

Matches buy and sell trades that are near-simultaneous, between accounts with possible common beneficial ownership, where the net change in delta and vega is effectively zero. Zero Greek change = zero economic substance = potential wash trade. Beneficial ownership table flags aggregated accounts per CBOE Rule 4.12 requirements.

-- DAS Gamma Squeeze Precursor · Estimates dealer gamma and retail call accumulation WITH dealer_gamma AS ( SELECT oi.symbol, oi.expiry, oi.strike, oi.option_type, oi.open_interest, g.gamma_per_contract, -- Dealers are net short customer-bought options -1 * (oi.open_interest * g.gamma_per_contract * 100) AS dealer_gamma_exposure FROM market.options_oi oi JOIN market.greeks g ON g.symbol = oi.symbol AND g.expiry = oi.expiry AND g.strike = oi.strike WHERE oi.option_type = 'CALL' AND oi.expiry <= CURRENT_DATE + INTERVAL '7 days' ), retail_flow AS ( SELECT symbol, SUM(call_volume) AS recent_call_vol, AVG(call_volume) OVER (PARTITION BY symbol ORDER BY date ROWS BETWEEN 30 PRECEDING AND 1 PRECEDING) AS avg_30d_vol FROM market.retail_options_flow WHERE date >= CURRENT_DATE - INTERVAL '2 days' GROUP BY symbol, date ) SELECT d.symbol, SUM(d.dealer_gamma_exposure) AS total_dealer_gamma, MAX(r.recent_call_vol / r.avg_30d_vol) AS retail_call_multiplier FROM dealer_gamma d JOIN retail_flow r ON r.symbol = d.symbol GROUP BY d.symbol HAVING total_dealer_gamma < -5000000 -- deeply negative dealer gamma AND retail_call_multiplier > 3.0 -- retail buying 3x+ normal pace ORDER BY total_dealer_gamma ASC;

Estimates aggregate dealer gamma exposure by assuming dealers are net short customer-bought calls. When total dealer gamma falls below −$5M per 1% move and retail call buying is 3x+ the 30-day average, the conditions for a self-reinforcing squeeze are present. This is the pattern that preceded the 2021 GME and AMC events.

-- DAS VIX SOQ Manipulation · Flags non-economic SPX orders during the VIX settlement window WITH soq_orders AS ( SELECT o.account_id, o.symbol, o.strike, o.option_type, o.quantity, o.side, o.status, o.ts, -- Estimated VIX contribution from this order (simplified Derman formula) (o.quantity * g.vega * 0.001) AS estimated_vix_impact FROM surveillance.order_flow o JOIN market.greeks g ON g.symbol = o.symbol AND g.strike = o.strike WHERE o.symbol = 'SPX' AND o.ts BETWEEN DATE_TRUNC('day', o.ts) + INTERVAL '9 hours 15 min' AND DATE_TRUNC('day', o.ts) + INTERVAL '9 hours 30 min' AND o.status = 'CANCELLED' -- cancelled = non-economic intent AND DAYOFWEEK(o.ts) = 4 -- Wednesday only (VIX expiry) ) SELECT s.account_id, COUNT(*) AS cancelled_soq_orders, SUM(s.estimated_vix_impact) AS total_vix_impact_pts, v.vix_futures_position AS pre_positioned_vix_contracts, (v.vix_futures_position * SUM(s.estimated_vix_impact) * 1000) AS est_manipulation_pnl FROM soq_orders s JOIN surveillance.vix_positions v ON v.account_id = s.account_id GROUP BY s.account_id, v.vix_futures_position HAVING cancelled_soq_orders >= 3 AND ABS(total_vix_impact_pts) > 0.3 ORDER BY est_manipulation_pnl DESC;

Detects accounts that: (1) submitted and immediately cancelled SPX options orders during the 9:15–9:30 VIX settlement window on expiration Wednesdays, and (2) held pre-positioned VIX futures that would have benefited from the resulting VIX settlement shift. The estimated manipulation P&L column is the key escalation metric for CFTC/SEC referral.

-- DAS Pinning Detection · Isolates underlying trades that exceed charm-model expectations WITH charm_expected AS ( SELECT symbol, SUM(charm * open_interest * 100) AS model_delta_change, pin_strike FROM surveillance.expiry_greeks WHERE ts >= DATE_TRUNC('day', NOW()) + INTERVAL '15 hours' GROUP BY symbol, pin_strike ), actual_flow AS ( SELECT symbol, SUM(CASE WHEN side = 'BUY' THEN quantity ELSE -quantity END) AS net_underlying_flow, STDDEV(price) AS price_vol FROM market.underlying_trades WHERE ts >= DATE_TRUNC('day', NOW()) + INTERVAL '15 hours' GROUP BY symbol ) SELECT c.symbol, c.pin_strike, c.model_delta_change AS charm_expected_shares, a.net_underlying_flow AS actual_shares_traded, (a.net_underlying_flow - c.model_delta_change) AS unexplained_shares, a.price_vol FROM charm_expected c JOIN actual_flow a ON a.symbol = c.symbol WHERE ABS(a.net_underlying_flow - c.model_delta_change) > 2.0 * ABS(c.model_delta_change) -- residual exceeds 2x model expectations AND a.price_vol < 0.002 -- price suspiciously pinned (low vol) ORDER BY ABS(unexplained_shares) DESC;

Computes the expected underlying trading from charm-driven delta rebalancing, then compares it to actual market flow. When actual trades exceed the charm model by more than 2x and price volatility is abnormally low near the pin strike, the residual is attributed to potential manipulation rather than legitimate hedging.

Deploy DAS

Terminal-native · macOS · Windows via WSL · Linux

1 — Install DAS and dependencies
brew install python@3.12 duckdb pip install das-surveillance anthropic websocket-client pandas pyarrow
2 — Configure credentials
das config --set CLAUDE_API_KEY=sk-ant-... das config --set CBOE_WS_TOKEN=your_token das config --set EXCHANGE_FEED=cboe,nyse,cme
3 — Start live surveillance
das start --mode live --patterns all --claude-analysis on # DAS now monitoring: CBOE · NYSE · CME · PHLX # SQL pattern engine running · Claude AI armed
DAS runs entirely local. Market data and surveillance findings never leave your machine. Claude analysis calls the Anthropic API over TLS — no surveillance data is retained by Anthropic beyond the API call.
1 — Enable WSL2 (PowerShell as Administrator)
wsl --install -d Ubuntu-24.04 wsl --set-default-version 2
2 — Install inside WSL2
sudo apt update && sudo apt install -y python3.12 python3-pip pip install das-surveillance anthropic websocket-client pandas pyarrow duckdb
3 — Configure and launch
das config --set CLAUDE_API_KEY=sk-ant-... das config --set CBOE_WS_TOKEN=your_token das start --mode live --patterns all --claude-analysis on
WSL2 runs a full Linux kernel alongside Windows. DAS inside WSL2 gets direct network access, full filesystem permissions, and native performance — identical to a Linux deployment. No emulation overhead for the SQL engine or WebSocket connections.
1 — Install dependencies
sudo apt install -y python3.12 python3-pip # Debian/Ubuntu pip install das-surveillance anthropic websocket-client pandas pyarrow duckdb
2 — Configure and launch as a systemd service
das config --set CLAUDE_API_KEY=sk-ant-... das config --set CBOE_WS_TOKEN=your_token das systemd install # installs das.service sudo systemctl enable --now das journalctl -u das -f # stream live surveillance logs
On Linux, DAS can run as a persistent systemd service with automatic restart on failure. Alerts are written to /var/log/das/alerts.jsonl — compatible with Splunk, Elasticsearch, or any SIEM that ingests JSON.
DAS uses DuckDB (embedded OLAP SQL), Anthropic Claude API, and exchange WebSocket feeds. Patterns reference: SEC Rules 10b-5, 13h-1, 15c3-5; CFTC CEA §4c(a)(1),(5), §9(a)(2); CBOE Rules 4.11, 4.12; FINRA Rules 2010, 4210, 5210; Dodd-Frank §747; MiFID II Art. 12-13.
